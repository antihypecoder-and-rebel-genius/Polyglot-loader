# Polyglot-loader
Polyglot Loader — это универсальный загрузчик плагинов для ExteraGram / AyuGram, который позволяет выполнять код на разных языках программирования внутри клиента Telegram. Вместо того чтобы писать плагины только на Python или Java, ты можешь использовать:

· Lua (лёгкий скриптовый язык) — временно вырезан. 
· Rust / Go / C++ (нативные плагины через .so) — для максимальной производительности и доступа к низкоуровневым API.
· Java (.dex) — для сложной логики с доступом к Android API.

Плагин работает как прокси: он регистрируется в системе как обычный Python-плагин, перехватывает системные вызовы (hooks) и перенаправляет их в загруженные скрипты или нативные библиотеки.

Ключевые возможности:

· Установка плагинов простым киданием файлов .lua, .js, .polyglot в любой чат.
· Автоматическое логирование через встроенный логгер или через плагин Log Overlay.
· Поддержка хука on_send_message_hook (будет расширяться).
· Возможность загружать нативные .so библиотеки без перезапуска клиента.
· Хранение скриптов в папке polyglot_scripts (не засоряет основную папку плагинов).

---

Как это работает (архитектура)

1. PolyglotNative — загружает libpolyglot.so (написан на C++). Эта библиотека умеет динамически линковать другие .so и вызывать экспортированные символы (например, polyglot_send_message_hook).
2. PolyglotLogger — собирает логи и отправляет их в log_overlay или в стандартный лог. Логируются все события: загрузка плагинов, вызовы хуков, ошибки.
3. PolyglotPluginManager — реестр загруженных плагинов. Знает, какой плагин на каком языке написан, и умеет вызывать его хуки.
4. Основной класс Plugin — наследуется от BasePlugin, регистрирует хуки на файлы и на отправку сообщений. При получении .lua/.js/.polyglot файла показывает диалог установки.
5. Хуки — через AndroidUtilities.openForView перехватываются попытки открыть поддерживаемые файлы, вместо этого запускается установщик. Через on_send_message_hook вызываются все зарегистрированные плагины.

Поток выполнения:

· Пользователь кидает test.lua в чат → нажимает «Открыть» → диалог «Install Polyglot Plugin».
· Плагин копирует test.lua в polyglot_scripts/test.lua и регистрирует его в менеджере.
· При отправке любого сообщения вызывается on_send_message_hook → менеджер ищет плагин test → вызывает Lua-функцию on_send_message_hook.
· Lua-функция модифицирует params.message и возвращает { strategy = 1 }.
· Polyglot Loader возвращает HookResult(MODIFY) → сообщение изменяется.
